# Security Policy

## Supported versions

Only the latest released minor version of TcxEditor receives security fixes.
Older versions are not patched. Please upgrade before reporting.

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |
| older   | :x:                |

## Reporting a vulnerability

If you believe you have found a security vulnerability in TcxEditor, please
report it privately by email to:

**jakubvonsyrek@gmail.com**

Please do **not** open a public GitHub issue for security reports.

Include in your report:

- A description of the issue and the potential impact.
- Step-by-step reproduction instructions (a sample `.tcx` file is ideal).
- Affected version / commit SHA.
- Any suggested mitigation, if you have one.

## Responsible disclosure timeline

- **Within 72 hours** &mdash; acknowledgement of receipt.
- **Within 7 days** &mdash; initial assessment and severity rating.
- **Within 30 days** &mdash; fix released or a public mitigation advisory
  published, whichever comes first.
- **Coordinated disclosure** &mdash; the reporter is credited (if they wish)
  in the release notes and `CHANGELOG.md` once the fix ships.

Thank you for helping keep TcxEditor and its users safe.
